DCI (Depuracao de Programas Baseada em Cobertura de Integracao -- Integration Coverage-based Debugging (ICBD))

Installation in Linux:

1. Run the command 'ant' in terminal

2. Create an environment variable 'DCI_HOME' to the path build directory of the project (e.g., export DCI_HOME=/home/<user>/dci/build)

3. Change the permissions for 'dci_unit' file in $DCI_HOME/bin/ (e.g., sudo chmod +x dci_unit)
 
4. Run 'dci_unit' file with the desired options: (e.g., $DCI_HOME/bin/dci_unit -r req -rt edge -hr tarantula -c src/)
-r: the folder that contains the coverage files generated by inss.
-rt: type of requirement (node, edge or dua, for while) 
-hr: type of heuristic used to calculate the suspiciousness (tarantula or drt, for while)
-c: path of the original classes of the subject program
-lt: type of list generated (by requirement, classes or packages) only for dci_unit

Ex:

#terminal 1
cd Dropbox/workspace/dci
ant
chmod +x ./build/bin/*

#terminal 2
cd Dropbox/experiments
export DCI_HOME=/home/higor/Dropbox/workspace/dci/build
$DCI_HOME/bin/dci_unit -r junit_method/ -rt node -hr tarantula -c src -lt requirement

1. To generate the new roadmap in xml, run the following command:

$DCI_HOME/bin/dci_mcp -r req/ -rt mcp -hr tarantula -c source/target/classes/ -lt requirement

It will generate the roadmap and the MCPs list.


2. To generate the new unit list with the method position attribute, run:

$DCI_HOME/bin/dci_rmcp -uf list_dci_NODE_TARANTULA_BY_PACKAGE.xml -rf list_dci_RMCP_TARANTULA.xml

It is necessary inform the old unit (-uf) list and the roadmap (-rf).
